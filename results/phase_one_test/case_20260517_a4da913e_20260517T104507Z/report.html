How To Read This Report

这份报告的目的不是展示漂亮结果，而是帮你定位项目问题。

Current Evidence Atomizer generates Evidence Atoms by LLM directly from items, clinical assertions, and EvidenceEventFrame nodes. It does not use coverage units and does not run dedup.

阅读顺序建议：先看 End-to-end Flow 和 Stage Diagnostics；再从 Raw Input、Stage Context、Clinical Sections、Structured Items 逐层往下查；最后到 Problem Dashboard 找重复出现的 warning/error code。
